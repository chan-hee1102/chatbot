# 요구사항 명세서 (SRS)

**서비스명** 개인 AI 어시스턴트 챗봇 (Web-based LLM Chatbot)
**문서 구분** Software Requirements Specification
**버전** v2.0
**작성일** 2026-05-12
**상태** 확정 (Baseline)
**참조 문서** [01_서비스기획안 v2.0](./01_서비스기획안.md)

---

## 1. 문서 정보

### 1.1 목적
본 문서는 웹 기반 LLM 챗봇 서비스의 기능 및 비기능 요구사항을 명세하며, 개발·테스트·운영의 공통 기준선(Baseline)을 확립합니다.

### 1.2 문서 규약
- 요구사항 ID는 `FR-NN`(기능) 또는 `NFR-NN`(비기능) 형식
- 우선순위: **Must** (필수) / **Should** (권장) / **Could** (선택)
- 각 요구사항은 검증 방법(Verification)을 명시

### 1.3 약어
| 약어 | 정의 |
|---|---|
| SRS | Software Requirements Specification |
| FR / NFR | Functional / Non-Functional Requirement |
| SSE | Server-Sent Events |
| JWT | JSON Web Token |
| TTFT | Time To First Token |
| RPM / RPD | Requests Per Minute / Per Day |
| WAL | Write-Ahead Logging (SQLite 저널 모드) |

## 2. 시스템 개요

### 2.1 시스템 컨텍스트
```
[Browser] ──HTTPS / SSE──> [Next.js on Railway] ──REST──> [Gemini API]
                                   │
                                   └──SQLite──> [Volume /app/data]
```

### 2.2 사용자 역할

| 데이터베이스 | SQLite, WAL 모드, 볼륨 마운트 |
| 외부 의존성 | Google Generative Language API |
| 클라이언트 | 최신 Chrome / Safari / Firefox / Edge |

## 3. 기능 요구사항 (FR)

### 3.1 인증

| ID | 기능 | 상세 | 우선순위 | 검증 방법 |
|---|---|---|---|---|
| FR-01 | 회원가입 | 이메일 정규식(`^[^\s@]+@[^\s@]+\.[^\s@]+$`) + 비밀번호 8자 이상 입력 시 계정 생성. bcrypt(saltRounds=12)로 해시 저장. 이메일은 소문자 정규화 후 저장 | Must | 정상 201, 형식 오류 400, 중복 409 |

| FR-03 | 로그인 | 이메일/비밀번호 검증 후 JWT(HS256, exp 7d) 발급. HttpOnly·SameSite=Strict 쿠키에 저장. 운영 환경에서 `Secure` 플래그 활성 | Must | 정상 200 + 쿠키, 실패 401 |
| FR-04 | 로그아웃 | 인증 쿠키를 `maxAge=0`으로 즉시 만료 | Must | 응답 200 + 쿠키 제거 |

| FR-06 | 자동 로그인(가입 후) | 회원가입 성공 시 동일 자격으로 즉시 로그인 처리 | Should | 가입 직후 `/chat` 도달 |

## 5. 데이터 모델

### 5.1 ERD
```
┌─────────────┐         ┌─────────────────┐         ┌─────────────┐
│   users     │ 1     N │  conversations  │ 1     N │   messages  │
│─────────────│─────────│─────────────────│─────────│─────────────│
│ id  (PK)    │         │ id  (PK)        │         │ id  (PK)    │
│ email (UQ)  │         │ user_id   (FK)  │         │ conv_id(FK) │
│ pwd_hash    │         │ title           │         │ role        │
│ created_at  │         │ created_at      │         │ content     │
└─────────────┘         │ updated_at      │         │ created_at  │
                        └─────────────────┘         └─────────────┘
                            ON DELETE CASCADE          ON DELETE CASCADE
```

### 5.2 테이블 명세

#### users
| 컬럼 | 타입 | 제약 | 설명 |
|---|---|---|---|
| id | INTEGER | PK, AUTOINCREMENT | 사용자 ID |
| email | TEXT | NOT NULL, UNIQUE | 로그인 이메일 (소문자 정규화) |
| password_hash | TEXT | NOT NULL | bcrypt 해시 |
| created_at | DATETIME | DEFAULT (datetime('now')) | 생성 시각 |

#### conversations
| 컬럼 | 타입 | 제약 | 설명 |
|---|---|---|---|
| id | INTEGER | PK, AUTOINCREMENT | 세션 ID |
| user_id | INTEGER | FK → users.id, ON DELETE CASCADE | 소유 사용자 |
| title | TEXT | NOT NULL, DEFAULT '새 대화' | 자동 생성 또는 사용자 지정 |
| created_at | DATETIME | DEFAULT (datetime('now')) | 생성 시각 |
| updated_at | DATETIME | DEFAULT (datetime('now')) | 마지막 활동 시각 |

#### messages
| 컬럼 | 타입 | 제약 | 설명 |
|---|---|---|---|
| id | INTEGER | PK, AUTOINCREMENT | 메시지 ID |
| conversation_id | INTEGER | FK → conversations.id, ON DELETE CASCADE | 소속 세션 |
| role | TEXT | CHECK(role IN ('user','model')) | 발신자 역할 |
| content | TEXT | NOT NULL | 본문 |
| created_at | DATETIME | DEFAULT (datetime('now')) | 전송 시각 |

### 5.3 인덱스
| 이름 | 대상 | 목적 |
|---|---|---|
| `idx_conversations_user_id` | `conversations(user_id)` | 사용자별 목록 조회 |
| `idx_messages_conversation_id` | `messages(conversation_id)` | 세션별 메시지 조회 |

### 5.4 PRAGMA 설정
- `journal_mode = WAL` — 동시 읽기/쓰기 향상
- `foreign_keys = ON` — Cascade 보장

## 6. API 명세

### 6.1 공통 응답 형식
모든 JSON 응답은 다음 형식을 따른다.
```json
// 성공
{ "success": true, "data": { ... } }

// 실패
{ "success": false, "error": "사람이 읽을 메시지" }
```

### 6.2 엔드포인트 목록

| Method | 경로 | 인증 | 응답 코드 |
|---|---|---|---|
| POST | `/api/auth/register` | × | 201 / 400 / 409 / 500 |
| POST | `/api/auth/login` | × | 200 / 400 / 401 / 500 |
| POST | `/api/auth/logout` | × | 200 |
| GET  | `/api/conversations` | ○ | 200 / 401 / 500 |
| POST | `/api/conversations` | ○ | 201 / 401 / 500 |
| DELETE | `/api/conversations/[id]` | ○ | 200 / 400 / 401 / 404 / 500 |
| GET  | `/api/messages/[id]` | ○ | 200 / 400 / 401 / 404 / 500 |
| POST | `/api/chat` | ○ | 200 text/event-stream / 400 / 401 / 404 |
| GET  | `/api/health` | × | 200 |

### 6.3 요청·응답 예시

#### POST /api/auth/register
```json
// Request
{ "email": "user@example.com", "password": "strongPw123" }

// Response 201
{ "success": true, "data": { "id": 1, "email": "user@example.com" } }
```

#### POST /api/auth/login
```json
// Request
{ "email": "user@example.com", "password": "strongPw123" }

// Response 200 (+ Set-Cookie: auth_token=...)
{ "success": true, "data": { "id": 1, "email": "user@example.com" } }
```

#### POST /api/chat (SSE)
```json
// Request
{ "conversationId": 12, "content": "오늘 날씨 어때?" }
```
```
// Response (Content-Type: text/event-stream)
data: {"text":"오늘은 "}

data: {"text":"맑고 화창합니다."}

data: {"done":true}

data: [DONE]
```

## 7. 외부 인터페이스

### 7.1 Google Gemini API

| 항목 | 값 |
|---|---|
| SDK | `@google/generative-ai` (0.24.x) |
| 모델 | `gemini-2.5-flash` (`GEMINI_MODEL`로 변경 가능) |
| 호출 방식 | `model.generateContentStream({ contents, systemInstruction })` |
| 컨텍스트 형식 | `{role: 'user'\|'model', parts: [{text}]}` 배열, 최대 20개 |
| 시스템 프롬프트 | KST 현재 시각 + 한국어 어시스턴트 페르소나 (요청마다 생성) |

### 7.2 외부 API 사용 제약
- Gemini 무료 티어: 모델별 RPM/RPD 한도 존재
- 한도 초과 시 HTTP 429 반환 → 클라이언트에 "AI 응답 생성 중 오류가 발생했습니다" 일반 메시지로 전달

## 10. 제약사항 및 가정

| # | 내용 |
|---|---|
| C1 | Gemini 무료 티어의 RPM/RPD 한도 내에서 운영. 초과 시 429를 사용자 에러로 표면화 |
| C2 | SQLite는 단일 파일 DB로 동시 쓰기 성능 한계 존재. 대규모 동시 사용자 환경에서는 PostgreSQL 전환 권장 |
| C3 | Railway 볼륨은 서비스 삭제 시 함께 사라지므로 주기적 백업 필요 |
| C4 | 컨텍스트 윈도우는 최근 20개 메시지로 고정. 매우 긴 대화는 모델이 초반 맥락을 놓칠 수 있음 |
| C5 | 시스템 프롬프트는 서버 시간 기준(KST)으로 주입. 클라이언트 시간대와 다를 수 있음 |

## 11. 인수 기준 (Acceptance Criteria)

본 시스템은 다음 시나리오가 모두 성공해야 인수된다.

| # | 시나리오 | 통과 기준 |
|---|---|---|
| A1 | 신규 가입 → 첫 메시지 | 가입 직후 `/chat`에 도달하고, 메시지 전송 시 2초 이내 첫 토큰 수신 |

| A4 | 마크다운 렌더링 | 코드 블록(언어별 색상), 표, 목록, 인용이 시각적으로 정확히 표시 |
| A5 | 응답 중단 | 스트리밍 중 중단 → 입력창 즉시 활성화 → 부분 응답 DB 저장 확인 |
| A6 | 대화 삭제 | 삭제 후 사이드바·DB에서 해당 세션과 메시지 모두 제거 |
| A7 | 권한 분리 | 사용자 A는 사용자 B의 대화 조회·삭제 시도 시 404 |
